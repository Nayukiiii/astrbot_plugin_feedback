# astrbot_plugin_feedback

AstrBot 意见反馈插件，将用户反馈收集到 [Teable](https://teable.io) 数据库，并支持管理员在 QQ 内完成审核、驳回、落地通知的全流程管理。

---

## 功能特性

- **用户提交意见** — 支持群聊和私聊，自动记录来源、时间、用户信息
- **实时通知管理员** — 新意见提交后立即私聊推送给管理员
- **管理员审核** — 支持批量通过 / 驳回，可附带驳回原因
- **功能落地通知** — 标记上线后自动通知原始提交者
- **群内意见征集** — 管理员可针对某条意见在原群发起公开征集，群友用「建议」命令参与，内容自动追加到 Teable 记录
- **功能落地计划** — 查看所有已排期的功能及预计上线时间
- **双管理员支持** — 可配置两个管理员 QQ

---

## 安装

在 AstrBot 管理面板的插件市场中搜索 `astrbot_plugin_feedback` 安装，或手动克隆到插件目录：

```bash
cd ~/AstrBot/data/plugins
git clone https://github.com/Nayukiiii/astrbot_plugin_feedback.git
```

---

## 配置

在 AstrBot 管理面板的插件配置中填写以下字段：

| 字段 | 说明 |
|------|------|
| `admin_qq_1` | 主管理员 QQ 号 |
| `admin_qq_2` | 副管理员 QQ 号（可留空） |

> Teable Token 和 Table ID 在 `main.py` 中硬编码，部署前请替换为自己的值。

### Teable 表结构

需在 Teable 中建立如下字段：

| 字段名 | 类型 |
|--------|------|
| 意见内容 | 文本 |
| 用户ID | 文本 |
| 用户昵称 | 文本 |
| 来源 | 文本 |
| 时间 | 文本 |
| 状态 | 单选（待处理 / 已通过 / 已驳回） |
| 驳回 | 勾选框 |
| 驳回原因 | 文本 |
| 落地 | 勾选框 |
| 预计落地时间 | 日期 |
| 功能备注 | 长文本 |

## 命令列表

### 用户命令

| 命令 | 说明 |
|------|------|
| `/意见 <内容>` | 提交一条意见或功能建议 |
| `建议 <内容>` | 在管理员发起的群征集中参与发言 |

### 管理员命令

| 命令 | 说明 |
|------|------|
| `拉取意见` | 获取所有待处理意见，生成编号列表 |
| `拉取已通过` | 获取已通过且设有预计落地时间的功能列表 |
| `批了 1` | 通过第 1 条意见并通知用户 |
| `批了 1,2,3` | 批量通过 |
| `驳回 1` | 驳回第 1 条意见 |
| `驳回 1,3 原因：重复` | 带原因驳回，并通知用户 |
| `上线 1` | 标记第 1 条功能已落地并通知用户 |
| `上线 1,2,3` | 批量标记落地 |
| `征集 1 [补充描述]` | 在该意见来源群发起公开征集 |
| `功能预计落地` | 展示功能落地计划时间表 |

## 工作流示例

```
用户：/意见 希望增加每日签到功能

→ Bot 回复：✅ 感谢你的反馈！你的意见已成功提交。
→ 管理员收到私聊通知

管理员：拉取意见
→ [1] 用户A  希望增加每日签到功能

管理员：批了 1
→ 意见状态更新为「已通过」
→ 用户A 收到通知：你提交的意见「希望增加每日签到功能」已被采纳 🎉

管理员：上线 1
→ 落地字段标记为 true
→ 用户A 收到通知：🎉 你提交的意见已正式落地，感谢你的贡献！
```
